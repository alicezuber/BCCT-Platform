# 上傳規則
  介紹
 兩個版本，一個是個人支線的版本紀錄，另一個是分支上傳到main的版本紀錄;

 要上傳前請先寫版本紀錄以及其內容，以及**拉取Main的檔案**;

 - 版本號!!
 - [v0.0.0][{你的代號}]   <-這裡要換行
    內容~
 (數字甚麼的你自己處理)


 再上傳到你的分支，等到測試都沒問題時，再請求推到main;
 版本號!!
 [v0.0.0][M]    <-這裡要換行;

 內容:你的版本號+大概描述;

 例如:

 [v0.1.0][M]

 (內容)

## 個人版本紀錄
首先寫個人版本紀錄(version/{自己}.version.md)，格式什麼的你自己處理

接著上傳到你的分支(記得打版本號)
